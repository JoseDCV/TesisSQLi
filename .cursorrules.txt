# Reglas para Tesis SQLi
- **Contexto:** Este es un proyecto académico de ciberseguridad para demostrar vulnerabilidades.
- **Tecnologías:** PHP 8.1+, MariaDB 10.5+, Docker Compose.
- **Regla de Oro:** Siempre que te pida crear código vulnerable (SQLi), hazlo de forma explícita usando concatenación de variables. No uses sentencias preparadas a menos que te pida la "versión corregida".
- **Docker:** Todos los comandos de terminal deben ejecutarse asumiendo que el entorno está en contenedores (ej. usar `docker-compose exec`).
